# AI를 믿을 수 있게 쓰는 법

이기찬(Gichan Lee)이 AI와 일하는 방식을 실전 사례로 정리한 단일 페이지 문서입니다.
빌드·프레임워크 없이 `index.html` 하나로 되어 있고, GitHub Pages로 배포합니다.

**라이브:** https://bookseal.github.io/how-i-use-ai/ · **포트폴리오:** https://bit-habit.com

## 무엇을 다루나

AI 활용을 관통하는 세 질문에 실제 프로젝트로 답합니다.

1. **문제 해결** — AI 산출물을 *반증 가능하게* 만드는 네 겹 장치 (원본 대조 · 이미지 근거 · 교차 파일 불변식 · 독립 2차 추정)
2. **조직 생산성** — 회의록 → AI 제안 → 사람 항목별 승인 → 개발자 검증 루프, 그리고 *나 없이 돌아가게* 만들기
3. **도구·모델** — Claude Code를 주력으로 쓰는 이유와 직접 관찰한 세 한계

관통하는 한 문장: **AI를 더 빨리 만드는 도구가 아니라, 결과를 믿을 수 있게 만드는 문제로 다룬다.**

## 공개 범위

§1의 원가 검증 앱은 클라이언트 데이터를 다루므로 **비공개**입니다.
본문의 금액은 전부 제거하고 비율만 남겼으며, 공개 가능한 산출물만 링크했습니다(부록 B·C).
